How to import existing PKG and AIFB files

Import PKG file

Sisar can import existing .pkg files into the .sisar project file. Once information contained in the .pkg file has been imported, no reference to the .pkg file will be stored.

To import a .pkg file, select Import PKG file... from the Tools menu and use the dialog to select the location of the .pkg file to open.

Import AIFB File

Sisar can import existing AIF builder project files (.aifb) into the .sisar project file. Sisar reads and uses the following information contained in the .aifb file:

  • Application UID

  • Extras bar captions for the application in different languages: the set of languages supported by the SIS component will be set based on this information. If the name of the SIS component has not already been defined for some of these languages, i.e the name is “”, the captions for those languages will be used to set the name.

Once information contained in the .aifbfile has been imported, no reference to the .aifb file will be stored.
